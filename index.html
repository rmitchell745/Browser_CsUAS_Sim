Browser CsUAS Sim

Single-file browser prototype for a component-system discrete event simulation. The current slice runs one hardcoded Red UAS against one Blue sensor / effector site, then pushes detections through tracking, classification, C2, firing, damage, logging, and Monte Carlo reporting.

    Version / Limitations

    This screen tracks what is solid in the current browser build, what remains intentionally limited, and which expert-user features are next.

    Current Version

    Browser CsUAS Sim: builder-first SME workflow pass with Scenario Wizard, Template Builder, grouped validation, report playback, Monte Carlo CSV, and structured export previews.

    Current Limitations

    No full terrain or environment builder, no reusable template library, no dedicated event timeline, no failure-driver table, and no canvas overlays yet for effector fields of view or intent headings.

    Coming Next

    Top candidates are event timeline review, failure-driver analysis, FOV / intent map overlays, richer environment authoring, and reusable local template workflows.

    Expert Workflow

    Use Scenario Wizard for quick starts, Template Builder for embedded template tuning, Report Viewer for outcomes, and Raw Data / Export for normalized scenario sharing and deep debug payloads.

    Model Constraints

    This tranche does not add new simulation physics. Ghost/clutter remain placeholders, TEWA payload estimation remains heuristic, and validation quality checks are advisory unless they reflect hard structural failures.
